# 拼豆图生成器（Pingdou）— 技术方案 v0.7

> 对应需求：`docs/01-需求文档.md` v0.7  
> 技术路线：stlite + Pyodide/WebAssembly + Cloudflare Pages 纯静态部署  
> 核心原则：用户图片不上传；所有算法本地执行；围圈交互服务真实拼豆场景，允许 8 字/叠边/共享边界，仅防圈套圈。

---

## 1. 总体架构

```text
Browser
├── index.html                  # 静态壳：loading、广告位、stlite mount
├── stlite runtime              # 自托管
├── Pyodide runtime             # 自托管 pyodide.js / wasm / stdlib
├── Python wheels               # 自托管 numpy / pillow / opencv / optional libs
├── Streamlit App               # main.py + pingdou package
└── Local algorithms             # crop/mask/quantize/label/render/export
        │
        ▼
Cloudflare Pages                # 仅静态文件，无后端 CPU
```

### 1.1 关键技术决策

| 模块 | 决策 | 原因 |
|------|------|------|
| UI | stlite / Streamlit | 快速搭建数据工具型 UI，Python 算法可直接复用 |
| Python Runtime | Pyodide WASM | 浏览器内运行 Python，图片不上传 |
| 图像处理 | Pillow + NumPy + OpenCV WASM | 裁剪、均色、轮廓、连通域本地完成 |
| 围圈 mask | OpenCV 原生轮廓/填充 | 避免 Pyodide 中 Python 循环慢；容错处理 8 字/叠边 |
| 标序 | `cv2.connectedComponents(connectivity=8)` | 避免引入 SciPy 大体积依赖 |
| 导出 | 手写 SVG XML + PDF 方案 | 优先矢量清晰，减少依赖体积 |
| 部署 | Cloudflare Pages | 静态 CDN、抗 DDoS、低成本 |
| 依赖加载 | 全量自托管 | 适配国内网络，避免外部 CDN 失效 |

## 4. 围圈 mask 技术细节

### 4.1 业务语义

真实拼豆用户只关心：哪些格子进入后续染色和标序。用户绘制不可能总是完美几何形状，因此：

| 形态 | 是否合法 | 结果 |
|------|----------|------|
| 单闭合圈 | 是 | 填充圈内 |
| 多个独立圈 | 是 | mask 并集 |
| 8 字圈 / 葫芦形 | 是 | 两个或多个封闭区域并集 |
| 相切/相交/共享边界 | 是 | 边界去重，区域并集 |
| 重复涂抹同一格 | 是 | 集合去重 |
| 外部毛刺 | 是 | 忽略，不报错 |
| 圈套圈 / 完全嵌套 | 否 | 拦截或提示一键合并 |

### 4.2 `mask_cv.py` API

```python
from dataclasses import dataclass
import numpy as np

@dataclass
class MaskError:
    code: str                 # NO_CLOSED_AREA / NESTED_LOOP / TOO_SMALL / UNKNOWN
    message: str
    cells: list[tuple[int,int]]

@dataclass
class MaskResult:
    mask: np.ndarray          # bool[H,W]
    boundary: np.ndarray      # uint8[H,W]
    errors: list[MaskError]
    warnings: list[str]


def build_mask(boundary_cells: list[tuple[int,int]], h: int, w: int) -> MaskResult:
    ...
```

### 4.3 推荐实现步骤

```python
def build_mask(boundary_cells, h, w):
    if not boundary_cells:
        return all_true_mask(h, w)

    boundary = rasterize_as_set(boundary_cells, h, w)

    contours, hierarchy = cv2.findContours(
        boundary,
        cv2.RETR_TREE,
        cv2.CHAIN_APPROX_SIMPLE,
    )

    nested = detect_nested_loops(contours, hierarchy, boundary)
    if nested:
        return MaskResult(mask=None, boundary=boundary, errors=nested, warnings=[])

    mask_a = fill_contours_union(contours, hierarchy, h, w)

    if not has_reasonable_area(mask_a, boundary):
        mask_b = fill_by_cv2_exterior_floodfill(boundary)
        mask = choose_better_mask(mask_a, mask_b)
    else:
        mask = mask_a

    mask = remove_external_spikes(mask, boundary)
    return MaskResult(mask=mask, boundary=boundary, errors=[], warnings=[])
```

注意：`fill_by_cv2_exterior_floodfill` 使用 OpenCV C++/WASM 原生 `cv2.floodFill`，不是旧版禁用的 Python 循环 flood fill。

### 4.4 圈套圈检测建议

1. 使用 `Hierarchy` 找 parent-child 关系。
2. 若 child contour 与 parent contour 没有共享边界/接触，且 child 完全位于 parent 内部，则判定为 `NESTED_LOOP`。
3. 若 child 与 parent 在栅格上接触或共享边界，按 8 字/葫芦形处理，不拦截。
4. 对用户提示时高亮内圈附近边界格，而不是显示复杂术语。

## 5. 点位色与量化

### 5.1 点位均色

- 输入：裁剪后图像 ndarray，网格 `grid_w × grid_h`，`mask`。
- 对每个有效格：取对应像素区域 RGB 均值；可配置中位数抗噪。
- 无效格：固定白色/透明，不参与后续 CCL。

### 5.2 色板格式

```json
{
  "palette_id": "mard_5mm_v1",
  "name": "Mard 5mm",
  "colors": [
    {"code": "A01", "name": "Black", "hex": "#111111", "sort_order": 1},
    {"code": "B03", "name": "Red", "hex": "#D62020", "sort_order": 2}
  ]
}
```

加载后预计算 Lab 值和亮度。

### 5.3 ΔE

- MVP：CIE76，速度快、依赖少。
- 后续：CIEDE2000，可作为高级精度开关。
- 允许限制最大颜色数，便于用户控制采购成本。

## 6. 分色标序与拼豆 UX

### 6.1 API

```python
def label_per_color(color_grid, mask, palette, prefix_style="short_letter"):
    """返回 labels_per_color, display_labels, blocks_per_color, prefix_map。"""
```

### 6.2 规则

1. 对每个色号独立生成二值图。
2. 用 `cv2.connectedComponents(binary, connectivity=8)` 求同色连通块。
3. 按 `(min_r, min_c)` 对块重排，保证从上到下、从左到右稳定。
4. 默认标签：`短字母前缀-局部号`，如 `A-1`。
5. 采购友好标签：`色板code-局部号`，如 `A01-1`。
6. UI 和导出必须显示图例：`A = A01 / Black / #111111 / 颗数 / 块数`。

### 6.3 自适应序号色

```python
def readable_text_color(hex_color):
    r, g, b = parse_hex(hex_color)
    luminance = 0.2126*r + 0.7152*g + 0.0722*b
    return "#E0E0E0" if luminance < 128 else "#303030"
```

深色豆必须浅字，浅色豆必须深字；这是打印可读性的 P0 要求。

## 7. 导出方案

### 7.1 SVG

- 直接拼 XML，避免引入重依赖。
- 格子用 `<rect>`，文字用 `<text>`。
- 网格线尽量合并为少量 `<path>`。
- 导出当前高亮状态时，未选色用低透明 fill，序号可隐藏。

### 7.2 PDF

优先级：
1. 浏览器打印 SVG 为 PDF（最快落地）。
2. 使用轻量 JS PDF 库或 Python PDF 库生成矢量 PDF。
3. 后续支持 A4/A3 分页、页码、重叠边距、裁切线。

### 7.3 CSV

至少包含两张逻辑表：

```csv
# colors
prefix,code,name,hex,total_beads,block_count
A,A01,Black,#111111,120,3

# blocks
label,prefix,code,count,min_r,min_c
A-1,A,A01,55,0,0
```

## 8. Streamlit/stlite 状态设计

```python
st.session_state = {
    "image_bytes": bytes,
    "crop": {"x": 0, "y": 0, "w": 100, "h": 100},

    "boundary_cells": set(),

    "prefix_style": "short_letter",
    "selected_colors": set(),
    "labels": {...},
    "exports": {...},
}
```

原则：重计算要按阶段缓存；边界每次变化只 invalidates mask 及后续结果，不必重读图片。

## 9. 性能与体积目标

| 项 | 目标 |
|----|------|
| 首次可见 loading | < 1s 显示 HTML/CSS |
| WASM 初始化 | 接受十几 MB 下载，但必须有进度与缓存 |
| 二次访问 | 大资源命中浏览器缓存 |
| 典型 80×80 mask | < 100ms |
| 典型 80×80 CCL | < 100ms |
| SVG 导出 | < 2s |
| 内存 | 避免保留多份大图；导出后释放中间对象 |

## 10. 安全、隐私与广告边界

- 不上传图片、不上传工程 JSON。
- 广告脚本不得读取用户图片 bytes、canvas 数据或工程状态。
- 可使用站点级统计，但需避免采集用户作品内容。
- 广告失败不得影响 app mount。
- CSP 后续应区分 app 静态资源、广告域名、分析域名。
