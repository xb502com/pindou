# 🧩 拼豆图纸生成器 Pingdou

> 把任意图片变成可制作的拼豆图纸。  
> **纯前端 / 静态网站 / 本地处理 / 不上传图片 / 完全开源。**

在线体验：**https://pindou.xb502.com/**

---

## ✨ 这是什么？

Pingdou 是一个面向拼豆、像素画、手工图纸制作的小工具。  
你可以上传一张图片，设置行列精度、颜色模式和最大颜色数量，然后生成带编号的方格图纸，方便照着摆豆子。

它的重点不是“做一张好看的预览图”，而是：

- 让每一个格子都是明确的方格；
- 让每一种颜色都有清楚的编号；
- 让相同颜色的豆子可以快速统计；
- 让图纸适合真实手工制作；
- 让用户不用担心图片隐私。

---

## 🔒 隐私说明：图片不会上传

本项目是**静态网站**，核心逻辑在浏览器里运行。

也就是说：

```text
你的图片 -> 你的浏览器本地处理 -> 生成图纸
```

不会走服务器上传流程，也不需要账号、数据库、后端 API。

所以你可以放心处理：

- 家人照片
- 宠物照片
- 手工作品参考图
- 自己的设计图
- 其它不想上传到第三方服务器的图片

## 🧠 核心功能

### 1. 图片转拼豆图纸

将普通图片采样成规则方格图，每一个格子对应一颗拼豆。

支持根据原图比例调整：

- 列数；
- 行数；
- 精细度；
- 固定方格比例。

生成结果不是随便拉伸图片，而是尽量保持每个格子都是标准方格，方便实际制作。

---

### 2. 彩色模式 / 黑白灰模式

生成图纸前可以选择：

#### 彩色模式

适合正常彩色拼豆作品。

可选择最大颜色数量：

```text
8 - 26 色
```

默认 8 色。  
如果原图颜色超过限制，会自动降低颜色精度，保证最终颜色数量不超过设定值。

#### 黑白灰模式

适合极简风、灰度图、头像、剪影、低成本制作。

会把图像映射为：

```text
黑 / 灰 / 白
```

然后按照同样的规则生成编号和统计表。

---

### 3. 拼豆色板适配

内置参考拼豆色板，使用类似：

```text
P01 / P18 / P03 / P04 ...
```

这样的色号体系。

颜色会尽量映射到真实可购买、可区分的豆子颜色上，而不是生成一堆现实中不好找的随机颜色。

---

### 4. 清晰的颜色编号规则

图纸中每个格子会显示类似：

```text
a1
b12
c103
```

含义是：

```text
a1 = a 颜色的第 1 个方块
b12 = b 颜色的第 12 个方块
c103 = c 颜色的第 103 个方块
```

其中：

- 字母代表颜色；
- 数字代表该颜色下的序号；
- 小写字母更简洁；
- 颜色相近的编号会尽量挨在一起；
- 统计表也会按颜色相近程度排序，方便找豆子。

---

### 5. 色区编码

同一种颜色如果是联通的，会按照色区进行编号。

规则：

- 八方向联通；
- 从上到下；
- 再从左到右；
- 单独一个方块也算一个色区；
- 不相连的同色区域会从前一个区域最后序号继续编号。

这样生成的图纸更适合人工阅读和制作。

---

### 6. 手动修色

有些图片在颜色边界处会出现不理想的自动判断，例如：

- 脸部边缘；
- 头发和背景；
- 阴影交界；
- 相近颜色过渡区。

本项目支持生成后手动点选格子修改颜色。  
修改后会自动重新计算编号和统计数量。

---

### 7. 围圈 / 局部图纸

支持在图片上圈出需要制作的区域。  
点一次标记，点第二次可以取消。  
嵌套圈也可以处理，以最外围圈为准。

适合只做头像、主体、局部图案，不一定要整张图都转换。

---

### 8. 导出与打印

支持：

- SVG 图纸；
- PNG 图片；
- CSV 统计；
- 浏览器打印 / 保存 PDF。

可以直接用于：

- 手机查看；
- 平板查看；
- 电脑打印；
- 保存图纸；
- 分享制作说明。

## 🌐 静态网站部署

部署时只需要把 `public` 目录里的内容放到网站根目录。

正确结构：

```text
/index.html
/assets/app/browser_app.js
/assets/palettes/basic.json
/assets/stlite/bootstrap.js
```

不要放成：

```text
/public/index.html
/public/assets/...
```

可部署到：

- Cloudflare Pages；
- GitHub Pages；
- Vercel；
- Netlify；
- Nginx；
- 1Panel 静态网站；
- 任意静态文件服务器。

1Panel / Nginx 场景下，把 `public` 目录内容上传到站点根目录即可。
